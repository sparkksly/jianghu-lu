# 武侠卡牌 Roguelike —— 战斗垂直切片设计 (v1 Vertical Slice)

- 日期: 2026-06-24
- 状态: 已批准设计,待写实现计划
- 引擎: Godot 4.6.1 stable
- 项目根: `RogueLike/new-game-project/`

## 1. 目标与范围

### 1.1 一句话

一个**武侠主题、连招驱动**的卡牌 Roguelike(爬塔结构)。核心创新是战斗:
双方**先把招式排到一条时间轴上(排招),再一起实时演算(观战)**,在演算中
读对对方意图、抢先手、打断前摇、靠攻防投相克与体力赌注,把没被拆掉的攻击
**滚成连段**。

### 1.2 为什么先做切片

完整游戏(全连招树 / 多流派 / 地图 / 商店 / 遗物 / 元进度 / Steam 集成)体量很大,
而本作的时间轴+体力+打断战斗是**全新机制,手感未经验证**。因此第一块只做一个
**战斗垂直切片**,把核心循环跑通、自测到「好玩」,再向外扩。手感验证优先。

### 1.3 切片【包含】

- **1 个流派:腿法**,8–12 张招式卡,含 ≥1 张格挡卡、≥1 张闪避(轻功)卡、≥1 张投技卡、≥1 张带「打断」词缀的招
- 完整的 **排招 → 观战** 单场战斗
- **离散时间轴**(每回合约 10 拍)+ **前摇/命中/后摇** 三段
- **体力系统**:每回合刷新、可排到 1.5x、实战体力奖惩、喘息破绽
- **攻 / 防 / 投三角** + **前摇打断(词缀化)** + **霸体(抗打断)词缀**
- **标签连招升格**(排招时,连续按序匹配配方 → 就地升格;支持同类与异类配方)
- **1 个 AI 对手**:会排招,并向玩家暴露**部分意图**
- 一场打完分胜负(HP 归零判负)

### 1.4 切片【不包含】(验证手感后再加)

爬塔地图、多流派、商店、遗物、元进度、美术打磨、音效、Steam 集成、存档系统、
多人 / 异步对战。

### 1.5 成功标准

- 能在 UI 中完成「排招 → 观战 → 分胜负」的完整一局
- 模拟器有单元测试覆盖核心规则(打断、打空惩罚、喘息、连招升格、攻防投相克)
- 自测时连招、打断、体力赌注三处**能带来有意义的决策**(主观,但这是切片要回答的问题)

## 2. 核心战斗机制

### 2.1 回合结构

一场战斗 = 若干**回合**。每回合两段:

1. **排招阶段**:玩家把手牌中的招式拖到时间轴的各拍上。体力槽刷新,可排到
   **体力上限的 1.5 倍**(超额部分=赌注,需靠实战体力奖励现付)。可见敌人**部分意图**。
2. **观战阶段**:双方时间轴**逐拍同时推进**演算,事件按实际发生兑现(命中 / 被闪 /
   打断 / 相克 / 体力奖惩 / 喘息)。回合结束后进入下一回合排招,直到一方 HP 归零。

- **HP**:跨回合,胜负线。
- **体力(Stamina)**:每回合刷新,是当回合的节奏 / 行动资源。

### 2.2 时间轴模型(离散拍)

- 一回合时间轴为 **N 个离散拍**(初值 N=10,可调)。离散便于精确模拟、单元测试与讲清规则。
- 每张招式在时间轴上占一个**连续区块**,三段:
  - **前摇 (startup)**:S 拍。蓄招阶段,**易被打断**。
  - **命中 (active)**:1 个命中拍。结算伤害 / 效果。
  - **后摇 (recovery)**:R 拍。收招阶段,被命中算「惩罚」。
  - 区块总长 = S + 1 + R 拍。
- 招式字段含**速度 / 优先级 (priority)**,用于同拍冲突时定先后。

### 2.3 打断(词缀化)

- 打断**不是通用规则**,而是招式词缀 **`打断 (interrupt)`**。
- 结算规则:
  - 攻击的**命中拍**落在对方**前摇**期间:
    - 攻击带 `打断` 词缀 → **取消对方该招(作废)+ 对方受硬直 + 体力罚**,并造成伤害。
    - 攻击**不带** `打断` → **照常造成伤害,但不取消对方的招**(对方该招仍会打出)。
  - 命中落在对方**后摇**:照常命中(惩罚收招),不取消未来招。
  - 同拍相撞:按 `priority` 决定先后;同优先级则判定为对拼(各自结算)。
- **`霸体 (super_armor)`** 词缀:带此词缀的招在其区块内**免疫打断**(仍会受到伤害,但不被取消)。
  是 `打断` 的天然克制,丰富读心层。**v1 包含**(实现成本仅为一处条件判断)。

### 2.4 攻 / 防 / 投三角

招式有**类型**字段:`攻击 / 防御 / 投技`。防御类下分两种行为:**格挡**与**闪避**。

- **攻击**:正常命中目标。**打空** = 命中拍落在对方的**闪避窗口**内(见下)或对方处于其他不可命中状态。
  打空时,尤其是**重击**,触发「用力过猛」:**额外体力惩罚 + 加长后摇**,自身陷入更长的可被惩罚窗口。
- **防御 · 格挡**:开一段**格挡窗口**(覆盖若干拍)。期间被**攻击**命中 → **抵消伤害 + 防御方回体力**。
  但**会被投技破防**(见下)。
- **防御 · 闪避(轻功类)**:开一段**短闪避窗口**。期间被**攻击**命中 → 该攻击**完全打空**(攻方触发用力过猛)。
  闪避不抵消、而是让对方落空,对**重击**尤其致命;代价是窗口窄、且不防投技。
  (格挡擅长吃连续轻攻击并回体力;闪避擅长惩罚单发重招。两种防御各有所长。)
- **投技**:
  - 命中拍落在对方**格挡窗口**内 → **破防,造成额外伤害**(投破防)。
  - 命中拍落在**未格挡**的对方 → 很弱 / 视为打空(盲投有风险)。
  - **攻击可打断投技的前摇**(若攻击带 `打断` 词缀;此处沿用 2.3 规则)→ 形成「攻克投」。
- 三角闭环:防克攻、投克防、攻克投(经由打断 / 时机)。

### 2.5 体力(赌注机制)

- 每回合体力槽刷新到上限 `STA_MAX`(初值待调)。
- 排招时招式体力消耗总和**最多可达 1.5 × STA_MAX**(超额规划允许)。
- 实战中体力**动态增减**:
  - **体力奖励**:命中 / 读对(如成功打断、成功格挡)→ 回体力。
  - **体力惩罚**:打空、被闪、重击落空(用力过猛)→ 扣体力。
- **喘息(破绽)**:演算中若某招执行所需体力不足(体力见底)→ 进入**喘息状态 K 拍**:
  - 该喘息区间内**已排好的招全部作废(跳过)**;
  - 喘息期间**受到的伤害有加成**(露出破绽)。
- 设计意图:超额规划 = 赌注。读对→体力奖励接济→打出超长华丽连段;读错→体力提前破产→喘息破绽巨罚。

> **平衡风险记录**:奖励与惩罚都挂在体力上,存在正反馈雪崩(赢者愈赢、崩者愈崩)的风险。
> 通过数值旋钮(奖励 / 惩罚幅度、喘息时长、STA_MAX、1.5x 系数)在调试期校准。模拟器须支持
> 无 UI 批量对打以做平衡测试。

### 2.6 标签连招升格(排招阶段)

- 招式带**标签**(如 `腿法 / 掌法 / 轻功 / 防御`)与类型(攻 / 防 / 投)。
- **连招配方**是一串**有序的「槽位条件」**,每个槽位可按以下任一方式匹配:
  - 按**标签**(如 `腿法`)
  - 按**类型**(`攻 / 防 / 投`)
  - 按**具体招式 id**
  - **任意 (wildcard)**
- 排招时,时间轴上若有**连续且按序**的一段招式满足某条配方 → **就地升格**为对应大招:
  占用更短、收益更高,但通常合并为**单一大命中点**(因此整套连招**只有一个被打断点,一旦被断全废**)。
- 示例配方:
  - 连环踢 = `[腿法, 腿法, 腿法]`(同类)
  - 佛山无影脚 = `[连环踢, 腿法, 腿法]` 或 `[腿法 ×5]`(同类,更高阶)
  - 乾坤大挪移 = `[攻, 防, 投]`(异类,按类型)
- 设计意图:不同卡只要满足槽位即可凑成同一连招 → 「发现路子」而非「背固定配方」。
  连招升格与超额体力赌注咬合:连招=高风险高回报。

## 3. 架构

### 3.1 核心原则:模拟与表现分离

**纯逻辑战斗模拟器**与**画面表现层**彻底分离。模拟器确定性、无节点依赖、可单元测试——
这是验证全新机制手感 / 平衡(可在无 UI 下批量对打)的关键。

### 3.2 模块

逻辑层(纯数据 / RefCounted,可测):

- `src/combat/combat_sim.gd` — 模拟器。输入「初始状态 + 双方排好的时间轴」,
  输出**确定性的战斗事件日志** `Array[CombatEvent]`。逐拍推进,无任何节点依赖。
- `src/combat/combat_state.gd` — 一局 / 一回合的状态(双方 HP、体力、时间轴、状态标记)。
- `src/combat/combat_event.gd` — 单个战斗事件(命中 / 打断 / 格挡 / 投破防 / 打空 / 喘息 / 体力变化…)。
- `src/combat/combo_rules.gd` — 连招配方表与匹配器(槽位条件 → 升格招)。
- `src/resources/move.gd` (Resource,**扩展现有 `src/resources/card.gd`**) — 招式数据:
  标签、类型(攻/防/投)、前摇/命中/后摇拍数、体力消耗、伤害、速度/优先级、
  词缀(`打断` / `霸体`)、关联升格配方。
- `src/ai/ai_planner.gd` — AI:生成一套排招时间轴;计算向玩家暴露的**部分意图**。

表现层(场景 / 节点):

- `src/scenes/plan_phase.tscn` + `plan_phase.gd` — 排招 UI:拖招上时间轴、显示体力槽与
  1.5x 上限、显示敌人部分意图、实时显示连招升格预览。
- `src/scenes/watch_phase.tscn` + `watch_phase.gd` — 观战:**回放**模拟器输出的事件日志。
  v1 用最简程序美术(色块 / 文字 / 进度条)即可。
- **复用**现有 `src/entities/combatant.gd`、`src/scenes/battle.tscn` 作为整合起点。

### 3.3 数据流

```
排招 UI ──(双方时间轴 Plan)──▶ combat_sim.simulate(state, plan_player, plan_ai)
                                        │
                                        ▼
                                Array[CombatEvent]  (确定性日志)
                                        │
                          ┌─────────────┴─────────────┐
                          ▼                             ▼
                   watch_phase 回放              单元测试断言 / 批量平衡测试
```

- 数据驱动:招式 / 配方均为资源 / 表,新增招式或连招**不改核心代码**。

## 4. 测试策略(TDD)

机制全新,模拟器**采用测试驱动开发**:用 GUT(Godot Unit Test)先写用例,再写实现。
必备首批用例:

- 攻击命中 / 未命中的伤害与体力变化
- 带 `打断` 词缀命中对方前摇 → 取消该招 + 硬直 + 体力罚
- 不带 `打断` 词缀命中前摇 → 仅伤害,招仍执行
- `霸体` 招免疫打断
- 重击打空 → 用力过猛(额外扣体力 + 加长后摇)
- 防御格挡攻击 → 抵消 + 回体力
- 闪避窗口使攻击打空 → 攻方触发用力过猛(重击额外扣体力 + 加长后摇)
- 投技命中格挡窗口 → 破防额外伤害;投空 → 很弱
- 攻克投(打断投技前摇)
- 体力见底 → 喘息 K 拍,区间内招作废 + 受伤加成
- 连招配方匹配(同类 / 异类 / 任意槽位) → 升格;升格招被打断 → 整套作废
- 模拟器对相同输入产出相同事件日志(确定性)
