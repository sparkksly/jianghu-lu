## 一、距离系统（核心新机制）

**数据**：`CombatState.distance: int`，`{贴身=0, 中=1, 远=2}`，**开局 = 中(1)**。

**步法改距离（新招式类型 STEP）**
- STEP 招带 `distance_delta`：上步 `−1`(拉近)、撤步 `+1`(拉开)。
- STEP 的效果在其**命中拍**（startup 结束、active 第一拍）落地。
- **同拍双方 STEP 先求和再应用一次**，然后 `clamp(0,2)`——一进一退抵消。

**攻击查距离（命中拍）**
- 每个攻击有 `range_min`/`range_max`（适用距离带）。
- 在 `_resolve_hit` 里，命中结算前先查：`range_min ≤ distance ≤ range_max`?
  - **否 → 够不着，走 `_whiff`**（落空 + 现有挥空体力惩罚，露破绽）。
  - 是 → 照常走 闪避/格挡/打断/命中 逻辑。
- **连招**：融合结果同样带 range，其**每段命中拍逐段查距离**（与普通攻击同一规则）。

**新轻量词缀（本切片需要）**
- **击退 `knockback`**：命中成功后 `distance = min(2, distance+1)`（把对手推开）。
- **点穴 `stun`(int 拍)**：命中后令对手 `gasp_until = t + stun`（复用现有"喘息=跳过排好的招"机制），被点穴期间跳招、且被打 +伤(现有 GASP_DAMAGE_BONUS)。

**距离不影响闪避**：闪身是纯时间窗口；距离只 gate"攻击够不够得着"。

**事件/可读性**
- 新增 `CombatEvent` 类型 `&"distance"`（amount=新距离）和 `&"reach"`（够不着，复用 whiff 的表现）。
- 观战时间轴标出距离变化（步法博弈可见）；排招卡牌标各招适用距离。

## 二、Move 结构新增字段（`src/combat/move.gd`）

- `enum Kind { ATTACK, BLOCK, DODGE, THROW, STEP }`（加 STEP）
- `@export var range_min: int = 0`、`@export var range_max: int = 2`（攻击适用距离带；默认任意）
- `@export var distance_delta: int = 0`（STEP 用）
- `@export var knockback: bool = false`（击退）
- `@export var stun: int = 0`（点穴拍数；0=无）

`CombatState` 新增：`distance: int = 1`（`clone()` 带上）。

**连招（`combos.gd` 不变配方，自动适配新招）**
- 腿法×3 → **连环踢**（弹腿/扫堂腿/侧踹 都是腿法）
- 连环踢 + 腿法×2 → **佛山无影脚**
- 攻 + 防 + 投 → **乾坤大挪移**
- 连招结果补上 range（腿法系 0-1）。

## 四、战斗模拟器集成（`src/combat/combat_sim.gd`，谨慎）

**本拍内顺序**：先结算 STEP 改距离，再用**更新后的距离**结算攻击命中（这样同拍的"上步+攻击"能让攻击落在带内）。

1. **STEP 结算**：tick 循环里，收集本拍双方"命中拍上的 STEP"的 `distance_delta`，求和后 `state.distance = clamp(distance+sum, 0, 2)`，发 `&"distance"` 事件。
2. **攻击查距离**：`_resolve_hit` 开头，若 `not (range_min ≤ distance ≤ range_max)` → `_whiff(...)`（够不着）并 return。
3. **击退**：命中成功后若 `atk.knockback` → `distance = min(2, distance+1)`，发 `&"distance"`。
4. **点穴**：命中成功后若 `atk.stun > 0` → `actors[defender].gasp_until = t + atk.stun`，发事件。
5. 其余逻辑（扣气/喘息/格挡/打断/优先）**不变**。

## 五、UI

- **排招卡牌**：在 `%d拍·%d气` 旁加距离标识(贴身/中/远 或图标)。STEP 招显示 `进/退`。
- **观战**：`CombatFeed` 给 `&"distance"` 一个标记（如"贴/中/远"或箭头），`&"reach"` 复用挥空表现；当前距离可在时间轴一角常显。

## 六、测试影响

- `move.gd`/`combat_state.gd`：新字段 + clone。
- **新增** `test_distance.gd`：STEP 改距离 + 同拍抵消 + clamp；攻击够不着→whiff；击退 +1；点穴→跳招。
- `test_content_fight.gd` / `test_combo.gd` / `test_balance.gd` / `test_ai`：新招替换后，连招配方仍成立（腿法×3→连环踢）；AI/balance 自适应（AI 需懂距离——见下）。
- `deck.gd` 改动 → 依赖 deck 的测试更新断言（move 名/数值）。

**AI**：本切片 AI 需最低限度懂距离——`ai_planner` 排招时，若想用某攻击，需保证命中拍距离合适（简单做法：开局插一个上步/撤步把距离调到自己主力招的带内，或只选 range 覆盖中距(1)的招）。细节留 plan。
