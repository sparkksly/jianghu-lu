## 一、北极星（整体愿景，仅作背景）

游戏是一个**武侠 roguelike**，暂名《江湖路》。核心身份：**一场"武学精进 + 江湖扬名"之旅**。
每一局是一个从**小渔村**打到**华山论剑**的江湖故事，分叉节点路线图，章节格局递增。

五根设计柱（已确认）：

1. **战斗（已有地基）** — 排招/观战时间轴对决，本 spec 即是夯实这根柱子。
2. **纵向成长：招式化繁为简（隐藏熟练）** — 基础招式常驻、绝学靠抽取；每招有**隐藏**熟练度，**低上限 + 绑定硬仗**，战胜强敌后以叙事事件提示"似有所悟"；练到位 → 前摇↓体力↓伤害↑、加霸体/debuff，**连招压缩成单卡**（多个腿法 8 拍 → 一张佛山无影脚）。
3. **横向成长：江湖奇遇** — 遇大师（学新招）、得秘籍（解锁进阶）、顿悟（花悟性/银两主动精进）。
4. **run 骨架：章节式江湖路线图** — 分叉节点（战斗/奇遇/精英/客栈/章节 boss），格局递增。
5. **江湖声望：善名/恶名/侠名** — 途中抉择累积声望，**目标深度=中**：分支路线 + 替换终局 boss，让每局是不同的江湖故事。

建造路线图（每片自成可玩、可验证的垂直切片）：

| 切片 | 内容 |
|------|------|
| **0/1（本 spec）** | **战斗地基**：拍/体力重分工 + 跨回合回气 + 反捉襟见肘约束 + 流派属性钩子 |
| 2 | 走通一条江湖路：节点地图 + 1 章 + 配班（基础常驻/领悟抽取）+ 战后学新招 |
| 3 | 化繁为简：隐藏熟练度 + 连招压缩 + 顿悟事件 |
| 4 | 声望分支：善恶侠名 + 分叉路线 + 替换终局 boss |
| 5 | 状态系统：中毒/内伤/流血/buff + 正邪机制差异（复用 `docs/wuxia-moves-research.md`） |

> 注：建造顺序上，"战斗地基"先于"走通江湖路"，因为体力跨回合经济必须先稳，否则上层 run 循环无法平衡。

## 二、本 spec 要解决的问题

当前战斗里 **「拍」和「体力」在重复劳动**，两者都在限制"你能出多少招"：

- **拍**：时间轴 `n_ticks`（当前 15）。招式占 `startup+active+recovery` 拍且不可重叠，所以拍数本身就限制了能塞下几个招。
- **体力**：`stamina`/`sta_max`，开招扣体力；`Plan.is_valid` 允许计划总花费至多 `floor(1.5*sta_max)`；且 [fight.gd](../../../src/scenes/fight.gd) **每回合把体力满血重置**。

问题：
1. 两条限制冗余，浪费设计空间。
2. **满血重置** → "气息悠长/内力深厚"的续航流派毫无意义（人人每回合满）。
3. 没有为"内功续航 vs 外功爆发"的流派轴留下属性钩子。

## 三、设计：拍与体力各管一件事

- **拍（定死 10，不多不少）= 时间窗口 / 密度。**
  职责：定时与排序——我这一掌哪一拍落、对手哪一拍出、谁先手、前后摇露不露破绽。
  10 拍物理上只编排得下约 3 个短招，**小而紧，逼你把每一拍用好**。拍是**软天花板**，不是出招数量的主预算。
- **体力 = 强度与续航。**
  职责：这几招是基础还是绝学、敢不敢爆发、**跨回合**能不能持续施压。流派身份活在这里。
- 二者咬在**不同维度**：**拍咬密度，体力咬强度与续航**。内力深厚 = 体力这一维更宽裕 = 更频繁打出高强度 / 铺满窗口。

### 现有 in-round 体力手感（保留不动）

以下机制已存在于 [combat_sim.gd](../../../src/combat/combat_sim.gd)，**本 spec 不改**，它们提供回合内的临场张力：

- 开招扣 `stamina_cost`；撑不住 → **喘息** `gasp_len` 拍（期间被命中 +`GASP_DAMAGE_BONUS` 伤害）。
- 命中 +1、格挡 +1、打断 +2、擒拿破防 +4、挥空 -2（重击 -4）、被打断踉跄 -2。体力 `clamp(0, sta_max)`。

## 四、机制改动（本 spec 的实现内容）

### 1. 数据层 `CombatState`（[combat_state.gd](../../../src/combat/combat_state.gd)）
- `stamina[i]`：改为**跨回合延续**，不再每回合重置；**地板恒为 0**（不允许负数）。
- 新增 `regen: Array[int]`：每回合开始回气量。起始值 ≈ 0.6–0.7 × `sta_max`（如 `sta_max=10` → `regen=6`）。
- `n_ticks`：**定死 10**。
- `clone()` 需带上 `regen`。

### 2. 回合开始回气（[fight.gd](../../../src/scenes/fight.gd)）
- 删除满血重置 `_state.stamina = _state.sta_max.duplicate()`。
- 改为：`stamina[i] = min(sta_max[i], stamina[i] + regen[i])`，对双方都生效。
- 首回合：`stamina` 初始化为 `sta_max`（开局气满）。
- `n_ticks` 由 14/15 改为 **10**。

### 3. 超额规则（[plan.gd](../../../src/combat/plan.gd) `Plan.is_valid`）
- 超额上限从"绝对 `floor(1.5*sta_max)`"改为相对**本回合当前气**：
  `total_cost() <= stamina_now + OVERCOMMIT_BUFFER`，其中 `OVERCOMMIT_BUFFER ≈ floor(0.3*sta_max)`。
  含义：可略微"借赌" in-round 回气（命中/格挡 +1），赌不中则喘息。
- `is_valid` 因此需要新增入参 `stamina_now`（替代当前以 `sta_max` 计算上限的写法）。
- **拍的边界检查保留**：`pm.start ∈ [0, n_ticks)` 且 `pm.end_tick() <= n_ticks`（招式不得探出 10 拍格子，今天已加）。

### 4. 表现层（[plan_phase.gd](../../../src/scenes/plan_phase.gd) / [plan_phase.tscn](../../../src/scenes/plan_phase.tscn)）
- 时间轴格子数 15 → **10**（`n_ticks` 传 10；网格宽度 `10 * TICK_W`）。
- 体力标签显示**本回合当前气** `current / max`（不再只显示 max），让"这回合甩不甩绝学"一目了然。
- `setup(...)` 多收一个 `stamina_now`，用于显示与超额计算。

### 5. 流派属性钩子（内功续航 vs 外功爆发）
- 本 spec **只建立钩子，不做流派内容**：流派差异完全由 `sta_max[i]` + `regen[i]`（及后续切片的招式花费档案）表达。
  - 内功续航：`sta_max` 高、`regen` 高 → 越打越稳、续航压制。
  - 外功爆发：招便宜但 `regen` 低、爆发后喘息重 → 先声夺人，怕拖。
- 起始默认双方对称（`sta_max=10, regen=6`）；非对称流派留给切片 2+。

## 五、反捉襟见肘的硬约束（验收标准）

> 用户红线："部分回气是对的，但不希望给玩家一种捉襟见肘只能挨打的体验。"
> 体力是**"气"——潮起潮落，不是单向流逝的血条**。

1. **永远出得起基础招**：`regen ≥ 2 × 基础招花费`（基础招 ≈ 2 体力，`regen=6` → 至少 3 个基础招）。任何回合都不会"一招都甩不出"。
2. **防守即回潮**：防守招（格挡/身法）命中后**净回气 ≥ 0**（格挡 +1 已满足）。低气回合打防守反击 = 回气 + 借力反打，而非干挨打。
3. **无死亡螺旋**：体力恒 ≥ 0，无深债；爆发的代价是"下一段无法连续爆发"，不是"被打废"。
4. **稀缺只在高档**：跨回合算计的是"甩不甩绝学/超不超额"，不是"还能不能动手"。
5. **拍为软顶**：10 拍只编排得下约 3 个短招；正常回合的决策是"把 2–3 招放在哪几拍最优"，而非"塞满"。

## 八、待调数值（实现时以这些为起点，后续平衡）

| 参数 | 起始值 | 备注 |
|------|--------|------|
| `n_ticks` | **10**（定死） | 时间窗口 |
| `sta_max` | 10 | 对称起手 |
| `regen`（每回合回气） | 6 | ≈0.6×池；满足"≥3 基础招" |
| `OVERCOMMIT_BUFFER` | 3 | ≈0.3×池 |
| 基础招花费 | 2 | 地板保证基准 |

### 后期密度待观察（用户提出，留待平衡）
10 拍在**前期**约 3 个短招是有意为之（逼你把每拍用好）；但**后期**招式变多变强后，3 招可能偏少。先记录，暂不改 `n_ticks`。设计上的预期解法是 **化繁为简（切片 3）本身就是密度的泄压阀**：招式练熟后占用的拍数变少（多腿法 8 拍 → 单卡数拍），于是**同样 10 拍内的有效行动密度随熟练自然提升**，而时间窗口仍保持 10 拍的紧凑博弈。若届时仍偏紧，再考虑：调短招 footprint、给特定流派/装备"额外拍数"词缀，或微调 `n_ticks`。

## 八点五、试玩修订（2026-06-25 当日）

实机试玩后调整（推翻了「拍定死 10」）：
- **拍 10 → 12**：10 拍配现有招式时长太挤——连环踢(原 9 拍)几乎要从第 0 拍开打、乾坤(11 拍)根本放不下。放宽到 12 拍。
- **前摇(startup)定为稀有属性**：基础招式 `前摇=0`，只有重踢这类缓慢招式保留前摇；于是「打断」也成为只对慢招有意义的稀有交互。基础招时长随之降到 2-3 拍（扫腿/轻踢 2 拍、身法 3 拍、格挡 4 拍、重踢 6 拍）。
- **UI 防误导**：排招卡牌从只显示气消耗 `轻踢(2)` 改为 `轻踢 / 2拍·2气`，明示招式占用的完整拍数。
- 新增回归测试 `test_starter_combos_fit_in_game_tick_budget`：保证可用连招(连环踢/乾坤)能装进当前 `CombatState.n_ticks` 并正常融合。
